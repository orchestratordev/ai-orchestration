Contributing to AI Orchestration

Thank you for your interest in contributing to AI Orchestration.

This project is built with a long-term engineering mindset. Every contribution should improve the platform while maintaining consistency, quality, and architectural integrity.

---

Before You Start

Please read the following documentation before contributing:

- README.md
- docs/INDEX.md
- docs/000-CONSTITUTION.md
- docs/001-ARCHITECTURE.md
- docs/006-DEVELOPMENT-STANDARDS.md

These documents define the engineering principles of the project.

---

Engineering Principles

All contributions should follow these principles:

- Simplicity First
- Readability Over Cleverness
- Documentation Before Implementation
- Security by Design
- Modular Architecture
- Long-Term Maintainability

---

Branch Strategy

Contributors should work on dedicated feature branches.

Examples:

feature/auth

feature/chat

feature/workspace

bugfix/login

refactor/database

Do not commit directly to the main branch unless authorized.

---

Commit Message Convention

Use clear and meaningful commit messages.

Examples:

feat(auth): implement login

fix(api): handle invalid requests

docs: update architecture

refactor(core): simplify registry

chore: update dependencies

---

Code Style

The project follows:

- TypeScript
- ESLint
- Prettier
- EditorConfig

Please ensure your code passes formatting before submitting.

---

Pull Requests

Every Pull Request should include:

- Purpose
- Summary of Changes
- Testing Performed
- Related Issue (if applicable)

Large Pull Requests should be divided into smaller logical changes whenever possible.

---

Testing

Before submitting changes:

- Verify functionality.
- Run available tests.
- Ensure no regressions are introduced.
- Update documentation if necessary.

---

Documentation

Documentation is part of the product.

If your contribution changes architecture, APIs, workflows, or user behavior, update the relevant documentation.

---

Code Review

Every contribution may be reviewed for:

- Architecture Compliance
- Readability
- Security
- Performance
- Documentation
- Maintainability

Requested changes should be addressed before merging.

---

AI-Assisted Development

AI tools may be used to assist development.

However:

- All generated code must be reviewed.
- Contributors remain responsible for code quality.
- AI-generated code must comply with project standards.

---

Respect the Architecture

Do not introduce structural changes without discussion.

Major architectural changes should be documented and approved before implementation.

---

Thank You

Every contribution helps improve AI Orchestration.

We appreciate your time, ideas, and commitment to building reliable software together.
