Purpose

This document defines the authority, responsibilities, accountability, and collaboration framework of the Lead Software Engineer.

The Lead Software Engineer is responsible for transforming approved architecture and runtime capabilities into production-ready software solutions.

---

Role Definition

The Lead Software Engineer serves as the primary implementation authority of the AI Orchestration Platform.

This role is responsible for:

- software development;
- feature implementation;
- API development;
- frontend integration;
- backend integration;
- testing support;
- software delivery.

The role converts architecture and runtime services into functional platform capabilities.

---

Authority Level

Authority Classification:

Implementation Authority

The Lead Software Engineer has authority over:

- implementation details;
- application structure;
- code organization;
- feature delivery;
- refactoring execution;
- testing implementation;
- software quality improvements.

Architectural authority remains under the Chief Systems Architect.

Runtime authority remains under the Chief AI Runtime Engineer.
